# Kubernetes概览

在Kubernetes环境中，DolphinDB主要包括以下几个部分：

  • DolphinDB实例： 每个DolphinDB实例对应一个Statefulset资源(一个Pod):
  • DolphinDB配置： DolphinDB的配置项定义在Configmap资源中(包括DolphinDB License)；
  • DolphinDB网络： DolphinDB的网络由Service资源进行代理;
  • DolphinDB存储： DolphinDB的存储主要由PVC资源进行管理，实现数据持久化。

在Kubernetes环境中，DolphinDB也可以通过CRD来定义，并由DolphinDB-Operator解析为Statefulset、Service和Configmap等Kubernetes原生资源。

提示

DolphinDB在Kubernetes环境中仍然需要使用License进行身份验证，DolphinDB在Kubernetes环境中的License以Configmap资源的形式存在，并挂载到DolphinDB所在的Pod资源中。

# 推荐环境

  • Kubernetes v1.18.0+在新窗口打开
  • Docker 20.10.10+在新窗口打开
  • Helm v3.0.0+在新窗口打开(可选)

注意

Helm在v3.0.0版本出现大规模变更，版本过低可能出现Chart包语法不兼容问题。
