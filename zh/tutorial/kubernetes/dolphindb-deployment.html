# 部署DolphinDB

DolphinDB可以运行在以Docker作为容器运行时的Kubernetes环境中。推荐环境如下：

  • Kubernetes v1.18.0+在新窗口打开        (参考部署Kubernetes环境)
  • Docker 20.10.10+在新窗口打开             (参考部署Docker环境)
  • Helm 3.0.0+在新窗口打开（可选）       (参考安装Helm工具)

注意

Helm工具在v3.0.0版本出现大规模变更，版本过低可能出现Chart包语法不兼容问题。

在Kubernetes环境中，可参考如下方式部署DolphinDB：

  • Kubernetes原生资源部署(基础): 使用Kubernetes原生资源部署DolphinDB;
  • CRD && Operator部署(进阶): 使用CRD + Operator的组合来部署Kubernetes环境中的DolphinDB.
  • DolphinDB MGR部署(推荐): 集成CRD + Operator、可视化界面、Prometheus、Grafana等丰富功能。

# Kubernetes原生资源部署

在Kubernetes环境中，可以使用StatefulSet等Kubernetes原生资源直接部署DolphinDB.

DolphinDB Chart包中封装了DolphinDB运行时所需的Kubernetes原生资源，可以使用Helm工具安装。详情可参考DolphinDB Chart安装。

# CRD && Operator部署

DolphinDB在Kubernetes中被定义为名为dolphindb(简称为ddb)的CRD，通过微服务组件DolphinDB Operator解析为相应的Kubernetes原生资源。

在安装有CRD && Operator的Kubernetes环境中，只需要通过简单的配置文件来部署ddb资源，就可以自动解析出相应的DolphinDB.

# DolphinDB MGR部署

DolphinDB MGR是DolphinDB在Kubernetes环境中的可视化管理平台，其集成了在Kubernetes环境中管理DolphinDB的所有功能，包含CRD && Operator、可视化界面、高可用方案、运维工具等。
